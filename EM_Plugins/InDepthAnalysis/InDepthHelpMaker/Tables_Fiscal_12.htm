Fiscal Tables

1.2. Number of taxpayers/beneficiaries

Description

The Table shows the total number (using weights) of units receiving/paying incomes, benefits or taxes.
The default unit is the household, but this can be changed: see Arguments.
The displayed incomes, benefits and taxes correspond to the items of Table 1.1., apart from Net budgetary effect, which is not displayed for Table 1.2.
Whether a unit is considered as a recipient/payer is by default determined by the respective income/benefit/tax taking a non-zero value for the unit. Again, this can be changed: see Arguments.

Arguments

The following arguments can be changed by the Fiscal settings dialogue:

  • Level of analysis: the unit to be counted for calculating the total number of units
    The amount of the income/benefit/tax, which defines whether the unit is a receiver/payer, is calculated as the sum over the members of this unit.
    Possible values are: Household or Individual or Individuals in household (also see General arguments / Level of analysis).
    Default: Household
  • Benefit/Pension receivers: a condition, on unit level, defining whether a unit is considered to be a benefit/pension receiving unit (also see General arguments / ‘Count unit’ conditions).
    The condition is applied on:
    • Total pensions
    • Total means tested benefits
    • Total non-means tested benefits
    and their components.
    Default: !=0
    Other Examples: >0; >ils_dispy*0.01
  • Other payers/receivers: a condition, on unit level, defining whether a unit is considered to be a taxpaying unit (also see General arguments / ‘Count unit’ conditions).
    The condition is applied on:
    • Total earnings
    • Other original income
    • Total taxes
    • Total SIC
    and their components.
    Default: !=0
    Other Examples: >0; >ils_origy*0.01
