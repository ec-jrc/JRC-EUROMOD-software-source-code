Changing the list of household types

The buttons alongside the Breakdown-list allow for adding, removing and ordering breakdown specifications.
Defining more than one breakdown specification multiplies the number of instances of the respective Table.
Thus, if the list specifies two breakdown specifications, for example for the default variable (Standard_HH_Categories) and for a user-defined household breakdown variable, the tool will generate two instances for each of the Tables 3.5. and 3.6.: one showing AROP rates/gaps for Standard_HH_Categories and the other showing AROP rates/gaps for the user-defined variable.
The order of appearance of the generated Tables in the Inequality and poverty Page will correspond to the order in the Breakdown-list.

Clicking the button Edit allows for specifying a list of Value descriptions, i.e. a set of descriptions that will be matched with the discrete values of the breakdown variable.
The descriptions will be displayed as row captions. If no description is filled in, the actual value is used.

Also see Inequality and poverty Tables / General arguments / Household type breakdown.
