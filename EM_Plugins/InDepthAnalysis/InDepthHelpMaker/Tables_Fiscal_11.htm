Fiscal Tables

1.1. Revenue-expenditure

Description

The Table shows, on an annual base, aggregate original income, government revenue and government expenditure.
In addition the last row displays the net budgetary effect, i.e. the change of a scenario's government budget (revenue minus expenditure) with respect to the reference scenario.

The following aggregates are displayed:

  1. Total earnings (ils_earns)
  2. Other original income (ils_origy - ils_earns)
  3. Total taxes (ils_tax)
  4. Total SIC (ils_sicdy + ils_sicer)
  5. Total pensions (ils_pen)
  6. Total means tested benefits (ils_benmt)
  7. Total non-means tested benefits (ils_bennt)
  8. Net budgetary effect (taxes + SSC - pensions - benefits)

The values are assessed as a weighted sum of the EUROMOD income list variables (indicated in brackets) in the respective baseline/reform output file.
Moreover, each aggregate, except from b and h, is decomposed into the single variables that build the respective income lists.

Some facts upon assessing the required information may be helpful to understand the table.

  • The information upon EUROMOD income lists is taken from the EUROMOD project indicated in the field EUROMOD project folder in the main settings dialogue.
  • The decomposition is omitted if no such information is available (e.g. if the country or system could not be identified in the EUROMOD project).
    The tool uses the EUROMOD log file, supposed to be found at the same path as the output file, to identify the system. If no log file is available, the tool tries to extract the system name from the output file name.
  • No decomposition is displayed, if more than one country is analysed.
  • In interpreting EUROMOD income lists, the tool accounts for EUROMOD extensions. The EUROMOD log file is used to assess the extension settings applied to generate the respective output file. If no log file is available, default extension settings are assumed.
  • The tool does not take into account whether the output was generated with add-ons or not.
  • The tool assumes ‘simple’ income list definition, i.e. that variables are either added (+) or subtracted (-). That means, that factors like +0.5 or -2 are ignored and just considered as added or subtracted, respectively.
  • Note that the sum of components and the value of the displayed sum (assessed via income list variables) will rarely be exactly equal. This is due to multiplying numbers in the output files, which by default have two decimal precission, with (possibly large) weights. Consequently, the preciseness of the match will increase with the number of decimals used for generating the output files.
    Nevertheless, as there are likely cases, where the sum of components and the total may not match due to reasons other than rounding (e.g. if no log file is available to assess extension settings), the tool performs a respective check and issues a warning if the percentage difference ((reform-base)/base) is larger than 0.00005.
  • A last note, summarising the advantages of the availability of a log file:
    • The output file can be renamed, as the log file works with file-codes instead of file names.
    • Finding the system name is not dependent on the output file having a standard name (i.e. cc_yyyy_std.txt).
    • Extension settings of the concrete run are taken into account.
