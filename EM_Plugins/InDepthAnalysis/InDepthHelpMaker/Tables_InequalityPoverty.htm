Inequality and poverty Tables

General arguments

Poverty lines

For Tables 3.3. AROP rates, 3.4. AROP gap, 3.5. AROP rates by type and 3.6. AROP gap by type users can define poverty lines by specifying two attributes:

  • type and
  • depending on type, either percentage or monthly amount
Users can select one of 3 types:
  • relative anchored: poverty line is calculated as a percentage (e.g. 60%) of the individual median of equivalised income in the baseline scenario
  • relative floating: poverty line is calculated as a percentage (e.g. 60%) of the individual median of equivalised income in the respective scenario
  • absolute (monthly): poverty line is indicated as a monthly amount
Income is defined in the respective application and equivalised using the modified OECD equivalence scale.

Household type breakdown

For Tables 3.5. AROP rates by type and 3.6. AROP gap by type users can define the household type over which the Tables are broken down.
For this purpose they must indicate a variable that exists in the baseline output file and provides a restricted number of values.
For example, the variable:

  • dlg (large family) will generate one row for households considered as large families (dlg=1) and one for the other households (dlg=0)
  • dms (martial status) will generate one row for each type of martial status (single, married, divorced, ...) of the first person in household (in the output file)
Note that the variable must not be monetary, because monetary variables are summed over household members.

By default a special template generated variable, Standard_HH_Categories, is used for the breakdown. This variable splits households into the following groups:

  • One adult <65, no children
  • One adult ≥65, no children
  • One adult with children
  • Two adults, <65, no children
  • Two adults, at least one ≥65, no children
  • Two adults with one child
  • Two adults with two children
  • Two adults with three or more children
  • Three or more adults, no children
  • Three or more adults with children
  • No adults (only persons <18)

Income definitions

For Tables 3.1. Inequality, 3.3. AROP rates and 3.4. AROP gap users can define income definitions, which reflect the path from original income to disposable income. For this purpose, they define:

  • a variable or formula for original income
  • a list of variables or formulas which define ‘add to previous’ values,
    where the first is ‘added’ to original income, the second is ‘added’ to this first, etc.
    In the end original income plus all ‘add to previous’ should yield disposable income.
    ‘Add to previous’ is in quotation marks because, in fact, users must define whether the value is added or subtracted by starting the formula with + respectively -.
In addition, users can define captions for original income, for each of the originating intermediate incomes and for disposable income.

By default the following list is displayed:

caption variable/formula
A = original income ils_origy
B = A - taxes and social insurance contributions (EQ_INC23) - ils_tax - ils_sicdy
C = B + pensions (EQ_INC22) + ils_pen
D = C + other benefits (disposable income, EQ_INC20) + ils_benmt + ils_bennt
