Fiscal Tables

1.4. Disaggregated concepts, number of units

Description

The Table shows total numbers (using weights) of units that fulfil a condition with respect to a specific variable.
By default the condition requests that the variable has a value different from zero.
The respective variables and conditions are defined in a specific table in the Fiscal settings dialogue.

Arguments
  • Concept: a name for the variable that will be displayed in the header column of the Table.
  • Variable/Formula: a variable or formula on individual level (see description of Table 1.3. for examples)
  • Formula filter: a condition on individual level (see description of Table 1.3. for examples)
  • Level of analysis: the level on which the Payers/Receivers condition (see below) is evaluated.
    Possible values are: Household or Individual or Individuals in household (also see General arguments / Level of analysis).
  • Payers/Receivers: a condition, on Level of analysis (see above), defining whether a unit is counted (also see General arguments / ‘Count unit’ conditions).
    Default: !=0
    Other Examples:
    • >500
    • <0
    • > yem*0.01 this means that, if ‘Level of analysis’ is ‘Household’ or ‘Individuals in household’, the unit is counted if the household value of ‘Variable/Formula’ is above 1% of household employment income.
    • !=0 && ils_dispy>0 in addition to only counting units with non-zero variable value, the tool does not count units with zero or negative disposable income.
    • !=0 && dag>70 in addition to only counting units with non-zero variable value, the tool does not count units, where the first person (in data) is 70 or younger.
