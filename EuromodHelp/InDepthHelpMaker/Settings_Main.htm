Main settings

EUROMOD project folder

This input box allows for defining the path of a EUROMOD project. By default, this is the currently loaded project.
The tool will try to load the information required for the analysis from this EUROMOD project. Most importantly this concerns the composition of income lists which is used for displaying the decomposition of revenue and expenditure variables/formulas in the Fiscal Tables.
When baselines and reforms are loaded (see Select baselines and reforms below) the tool will check whether the respective output files were generated with the indicated EUROMOD project, by using the EUROMOD log file if it exists, and issue a warning if not. The tool will still try to extract the required information by matching the filename to a country/system within the selected project. If retrieving the info is not possible (if for example there is no country or system that seems to correspond with an output file), the tool will, apart from issuing a warning, not display any decomposition of revenue and expenditure variables/formulas in the Fiscal Tables.

Selecting baselines & reforms

Clicking the button Select baselines & reforms opens a special file-selection dialogue that allows for selecting the baseline and reform output files used for the analysis. If the selection is successful, the files are shown in the two tables above the button.
The upper table shows information about the selected baselines. Usually there will be one baseline, but as the tool allows for multi country mode, more than one baselines may be shown, with the restriction that there cannot be more than one baseline per country.
The lower table shows information about the selected reforms. There should be one or more reforms per baseline.
The columns of the tables show the following information:

  • The first column shows a check or x, depending on whether the file exists and is usable.
  • The second column shows the name of the output file. Moving the mouse over this name shows a tooltip with the path of the output file.
  • The third column allows for defining a custom label for this baseline/reform, that will be used in the generated Tables. If the label is left empty, the system name will be used by default. If the system has a standard name, e.g. MT_2020_std, a ‘pretty’ version is displayed, e.g. Malta 2020.
Note that the selected baselines and reforms are remembered for the duration of the EUROMOD session, but not saved in the settings (see Administrate settings below).

Selecting Pages/Tables for calculation

The table on the right side of the dialogue lists all the Tables that can be generated by the tool. The list is grouped into three categories:

  • Fiscal
  • Distributional
  • Inequality and poverty
This correspond with the structure of the Tables in the In-depth, where there are three Pages, one for each category, containing the Tables listed under this category.
Consequently deselecting a category results in showing none of the Tables of the category (as the Page is not displayed). In addition, single Tables can be deselected from being displayed (in a displayed Page).

Changing comparison mode

By default all comparisons in the Tables refer to comparing (any) reforms with the baseline. This can however be changed by ticking Compare with previous scenario. If ticked, only the 1st reform is compared with the baseline, all following reforms are compared with their predecessor. The order of reforms is determined by the order in which they are listed in the reforms table, which is defined by their respective selection in the file-selection dialogue.

Saving a merged dataset

Ticking the checkbox Save merged dataset will, once the Run button is clicked, generate one file per baseline that contains the content of the baseline and corresponding reform output files, merged by using the variable idperson.
The input box below this checkbox allows for defining the folder where these merged datasets will be saved. The filename of the merged dataset is composed automatically by using the country short name and all system names (for example mt_2017_2018_2019.txt).

Changing settings for Fiscal/Distributional/Inequality & poverty Tables

Selecting a sub menu of the Advanced settings menu opens a dialogue, which allows for defining the settings, and therewith the concrete content, of the Tables displayed by In-depth. For details see Fiscal settings, Distributional settings and Inequality & poverty settings.
Moreover, the sub menu Reset settings allows for restoring the default settings for all Tables.

Administrating settings

The tool allows for generating settings files which store the current settings for reuse in later sessions. The content of such a settings file consists of

  • the selection of Pages/Tables (as shown in the table on the right side of the dialogue)
  • the settings defined in the Fiscal/Distributional/ Inequality & poverty settings dialogues
  • the path defined in the input box Save merged dataset
  • whether the merged dataset should be created once Run is clicked, or not
  • the path defined in the input box EUROMOD project folder
  • whether comparisons should refer to the baseline or the previous reform
  • the paths of baseline and reform output files as selected in the file-selection dialogue
The selected baseline and reform output files (as displayed in the tables on the left side of the dialogue) are not stored in the setting files.


When the tool is first loaded during a EUROMOD user interface session, it shows default settings.
The sub menus of the File menu allow for

  • loading any existing settings file with the sub menu Load.
  • storing the current settings with the sub menu Save. If this menu is clicked for the first time during a session, it will ask for the path and filename for the settings file. On each further click it will use this selection.
  • storing the current settings with the sub menu Save As always asks for the path and filename for the settings file.
  • resetting and starting a new settings file with the sub menu New.
Note that the settings as well as the baseline and reform output files are kept in memory during the EUROMOD user interface session, meaning that, if the tool is closed and reopened it still shows all the user choices as before the closing.
