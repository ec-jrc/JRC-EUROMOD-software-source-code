Fiscal Tables

1.3. Disaggregated concepts

Description

The Table shows aggregate weighted values of user-defined variables on an annual base.
The respective variables are defined in a specific table in the Fiscal settings dialogue.

Variables definition

Each variable displayed is defined by the following arguments:

  • Concept: a name for the variable that will be displayed in the header column of the Table.
  • Variable/Formula: a variable or formula on individual level (also see General arguments / Formulas).
    Examples:
    • yem
    • ils_earns
    • ils_earns - ils_earns_base
    • yem + yse
    • dgn questionable, as it calculates the weighted number of women times 12, as the Table operates on annual base
    • dag questionable, as it calculates the weighted sum of age times 12
    • (yem + yse) / ils_origy if used without a ‘Formula filter’, will most likely yield NaN (not a number), as it generates divisions by zero for some individuals, which prevents building the total.
      Moreover it does not make much sense, as it calculates the weighted sum of a percentage times 12.
    • (yem / ils_tin) * tin_s may make sense, but still requires a ‘Formula filter’ to avoid yielding NaN for the total.
  • Formula filter: a condition on individual level (also see General arguments / Conditions).
    The filter allows for restricting calculations to specific population groups and may be required to avoid divisions by zero.
    Examples:
    • dgn == 1
    • dag>18 && dag<70
    • ils_dispy>500 note that this refers to individual disposable income
    • ils_tin != 0
