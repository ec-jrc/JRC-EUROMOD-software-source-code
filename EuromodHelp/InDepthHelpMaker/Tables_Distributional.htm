Distributional Tables

General arguments

Breakdown

All Distributional Tables show results for the whole population, as well as broken down over an ‘argument’ splitting the population into groups.
There are two ways to define this argument:

  • by a variable with a restricted number of values
    For example:
    1. the variable dgn (gender) will generate two rows, one with results for men and the other with results for women
    2. the variable drgn1 (region nuts level 1) will generate one row with results for each region
  • by quantiles of a variable
    For, example:
    1. the variable ils_dispy (disposable income) together with the option quantiles set to 5 and the option equivalised to yes, will generate five rows, one with results for each quintile of equivalised individual disposable income
    2. the variable yem (employment income) together with the option quantiles set to 2 and the option equivalised to no, will generate two rows, one with results for the units with employment income below the median and the other with results for the units with employment income above the median

Note that the breakdown always refers to variables of the baseline. There are five default breakdowns: Disposable_Income_Deciles, Standard_Labour_Categories, Standard_HH_Categories, Standard_Gender_Categories, Standard_Age_Categories. For more details on the default breakdowns see the Default Breakdowns page.

Also note that, if there are no units taking a certain value of a variable, no row is displayed for this value. If, for example, decile 6, does not have any members in the selected population, the row for decile 6 is omitted.

See Distributional settings / Changing population breakdown for instructions on how to define breakdowns in the settings dialogue.

Target population

For all Distributional Tables the respective results can be restricted to a specific target population (separately for each Table).
For a description and examples on how to define the respective condition see General arguments / Conditions.
Note that the condition is evaluated at the Level of analysis selected for the Table.
