Summary of common parameters

parameter name value type compulsory / optional default value description
output_var variable compulsory [1] n/a variable for storing the result of the function, result of function overwrites current value
output_add_var variable variable for storing the result of the function, result of function is added to current value of output variable
result_var variable optional n/a variable for storing the result of the function, always overwritten
who_must_be_elig categorical optional nobody

function’s calculations are carried out if …

  • one_member (or one): one member of the assessment unit is eligible
  • one_adult: one adult member of the assessment unit is eligible
  • all_members (or all or taxunit): all members of the taxunit are eligible
  • all_adults: all adult members of the taxunit are eligible
  • nobody: always

"eligible" is determined by the variable indicated by parameter elig_var
elig_var variable optional sel_s variable indicating whether a person is "eligible"

zero: person is not eligible

non zero: person is eligible
lowlim formula optional neg. ∞ [3] replaces result of function if result is smaller
uplim formula optional pos. ∞ replaces result of function if result is higher
threshold formula optional neg. ∞ replaces result of function if result is smaller:

if lower limit is not defined by zero, otherwise by lower limit
limpriority categorical optional upper

if upper limit is smaller than lower limit:

upper: upper limit dominates

lower: lower limit dominates

not defined: a warning is issued (and default applied if ignored)

round_to

round_up

round_down

 value optional n/a result is rounded to nearest whole number if set to 1, to nearest number with 1 decimal if set to 0.1, to nearest 10 if set to 10, etc.

_up always rounds up, _down always rounds down
TAX_UNIT taxunit compulsory [2] n/a assessment unit for function’s calculations
"switch" categorical compulsory n/a

not literally a parameter

  • on (or 1): function is switched on
  • off (or 0): function is switched off
  • toggle (or 2): function is initially switched off, to be switched
    on later (parameters of the function are read and checked)
run_cond condition optional {1}

conditional switch: function is only carried out if the condition is fulfilled

the condition should not be individual or household based, but refer to a specific processing state or other global condition
#_lowlim footnote optional neg. ∞ replaces operand if operand is smaller (see EUROMOD Functions - Footnote parameters for the further specification of operands)
#_uplim footnote optional pos. ∞ replaces operand if operand is higher (see EUROMOD Functions - Footnote parameters for the further specification of operands)
#_amount footnote optional n/a indicates numeric value of an operand (see EUROMOD Functions - Footnote parameters for the further specification of operands)
#_level footnote optional n/a indicates alternative assessment unit of an operand (see EUROMOD Functions - Footnote parameters for the further specification of operands)
#_xxx footnote optional n/a indicates further specification of a query (see EUROMOD Functions - Queries)
#_type footnote optional n/a explicitly indicates type of an operand as var (or variable), il (or incomelist) or query (see EUROMOD Functions - Footnote parameters for the further specification of operands)

[1] for policy functions with exceptions

[2] for policy functions

[3] ∞ in fact stands for 999999999999.99.
